# renderField
`renderField(data: RenderFieldPayload | RenderFieldPayloadAsFunction): React.ReactNode`

With *renderField* you will be able to render a certain customizable field, which was defined in your form settings.
Below are its properties, it is worth highlighting that all the properties of *[register](https://react-hook-form.com/docs/useNiceForm/register)* presented in react-hook-form
are included here. In order to save time, we will focus on the extra properties.

<div className="subheading">
    definitions
</div>
<table>
  <tr>
    <th>Property</th>
    <th>Type</th>
    <th>Description</th>
  </tr>
  <tr>
    <td>[type](#basicExample)<span className='required'/></td>
    <td>`string`</td>
    <td>
        It informs the type of field, it will serve as a basis for knowing which component to render.
    </td>
  </tr>
  <tr>
    <td>[name](#basicExample)<span className='required'/></td>
    <td>`string`</td>
    <td>
        Informs the name of the field, used to distinguish one field from another.
    </td>
  </tr>
  <tr>
    <td>[active](#active)</td>
    <td>`boolean`</td>
    <td>
        Determines whether a field is active or not on the form.

        If `true`, the field will be integrated into the form.

        If `false`, changes to them or the form will have no effect.

        Default: `true`
    </td>
  </tr>
  <tr>
    <td>[dependsOnToClear](#dependsOnToClear)</td>
    <td>`string | string[]`</td>
    <td>
        Determines which field or fields it depends on. 
        If the field on which it depends changes its value, 
        it will be cleared in response to the action.
    </td>
  </tr>
  <tr>
    <td>[input](#input)</td>
    <td>`input(value: any): any`</td>
    <td>
        Computes the field's input value. It is recommended to reference this field in `initialValues`.
    </td>
  </tr>
  <tr>
    <td>[output](#output)</td>
    <td>`output(value: any): any`</td>
    <td>
        Computes the field's output value.
    </td>
  </tr>
  <tr>
    <td>[watchChangesOnlyPrimitiveTypes](#watchChangesOnlyPrimitiveTypes)</td>
    <td>`boolean`</td>
    <td>
        Determines how the field undergoes some type of mutation in relation to changes in its properties, by default it will listen for mutation only for primitive values.
      
        Default `true`
    </td>
  </tr>
  <tr>
    <td>[debounceTime](#debounce)</td>
    <td>`number`</td>
    <td>
        Determines a delay time for field value change feedback.
        
        Default: `400ms`
    </td>
  </tr>
  <tr>
    <td>[enableDebounce](#debounce)</td>
    <td>`boolean`</td>
    <td>
        Enable debounce for this field.
        
        Default: `false`
    </td>
  </tr>
  <tr>
    <td>[validate](#validate)</td>
    <td>`Record<string, Validate<any, FieldValues>>`</td>
    <td>
        react-hook-form specific validation property.
    </td>
  </tr>
</table>
